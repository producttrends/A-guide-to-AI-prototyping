Как я создала это в Claude - пошагово:

1

Загрузите картинки

Если у вас есть свои изображения или скриншоты дизайна, загрузите их на imgbb.com. После загрузки скопируйте прямую ссылку на изображение (правый клик → "Копировать адрес изображения").

2

Первый промпт - базовая структура

Отправьте скриншот в Claude с промптом:

Build a prototype to match this Airbnb design. Match it exactly. Use React with TypeScript and Tailwind CSS. Include the search bar with location, dates, and guests inputs.
3

Добавьте фильтр цен

После создания базовой структуры:

Create a new PriceFilter component that: - Shows a price range input when clicked - Has minimum and maximum price inputs - Displays an overlay that covers the page when active - Updates the display text when prices are selected - Closes when clicking outside the popup
4

Создайте слайдер

Для интерактивного выбора цены:

Create a reusable Slider component and integrate it into the PriceFilter. The slider should have: - Blue track for the filled portion - Black circular handle - Smooth dragging functionality - Click anywhere to set value - Connect to update the minimum price value
5

Добавьте меню пользователя

Для полноты интерфейса:

Create a UserMenu component that: - Shows/hides the dropdown on click - Closes when clicking outside - Includes login, signup, and other menu options - Uses Lucide icons for menu items - Has proper hover effects and spacing - Matches Airbnb's design with white background and shadow
6

Финальные штрихи

Добавьте анимации и улучшения:

Add smooth transitions and animations: - Fade in/out for overlays - Slide animations for dropdowns - Hover states for all interactive elements - Loading states for search results Make it responsive for mobile devices

💡 Секрет успеха:

Чем подробнее указывать компоненты, тем точнее будет результат.
Разбивайте задачу на маленькие части, описывайте каждый элемент интерфейса, указывайте конкретные библиотеки и фреймворки. Claude понимает контекст и создаёт именно то, что вы описали.

Решение проблем в Claude без знания кода

🤔

Reflection

Заставьте Claude сначала создать план, а не писать код. Попросите объяснить возможные причины ошибок.

📦

Batching

Разбивайте задачи на маленькие части. Claude лучше справляется с пошаговыми инструкциями.

🎯

Be Specific

Указывайте конкретные файлы, компоненты, строки кода. Чем точнее инструкции, тем лучше результат.

💾

Lost Context

Claude сохраняет версии в Artifacts. Можете вернуться к предыдущей версии в любой момент.

🔍

Error Analysis

Копируйте точный текст ошибки в Claude. Он проанализирует и предложит решение.

Debugging Example
I'm getting an error when clicking the submit button. Don't write any code yet. First, think through 3-4 possible reasons why this error might be occurring. After explaining the possible causes, suggest which one is most likely and how to verify it. // Claude отлично понимает контекст и поможет найти решение!

От идеи к сайту с Claude за минуты

1

Подготовьте идею

Опишите что хотите создать или покажите скриншот дизайна. Claude понимает и русский, и английский язык. Можете просто сказать "сделай сайт как Airbnb".

2

Отправьте промпт в Claude

Скопируйте один из шаблонов выше или напишите свой. Приложите скриншоты если есть. Claude сразу покажет результат в Artifacts.

3

Итеративная разработка

Начните с core functionality. Добавляйте фичи по одной. Тестируйте после каждого изменения.

4

Тестирование с пользователями

Поделитесь ссылкой на прототип. Соберите фидбек. Итерируйте быстро.

5

Экспортируйте готовый сайт

Нажмите кнопку "Publish" в Artifacts для получения ссылки. Или скачайте код и выложите на GitHub Pages. Весь процесс занимает минуты!

Как опубликовать сайт из Claude

1

Создайте репозиторий

Зайдите на github.com, нажмите "New repository". Назовите его (например, "my-prototype"), сделайте публичным

2

Экспортируйте код из Claude

В Claude Artifacts нажмите на три точки → Download. Или используйте кнопку Publish для мгновенной публикации.

3

Загрузите файлы

В репозитории нажмите "uploading an existing file", перетащите файлы, нажмите "Commit changes"

4

Включите GitHub Pages

Settings → Pages → Source: Deploy from branch → Branch: main → Save. Через 2-3 минуты сайт будет доступен

5

Альтернативы для React

Vercel: vercel.com - автоматический деплой из GitHub
Netlify: netlify.com - drag & drop деплой
Cloudflare Pages: pages.cloudflare.com - быстрый CDN

30 советов от Y Combinator по созданию сайтов с AI

🎯 Советы #1-3: Планирование
Не ныряйте сразу в код! Сначала создайте детальный план с AI, сохраните его в markdown файл и реализуйте по секциям. Помечайте сложные фичи как "не буду делать".
🛠️ Советы #4-6: Выбор инструментов
Новички: начните с Claude или Replit. Опытные: добавьте Windsurf, Cursor. Совет: используйте Claude для быстрых прототипов, а другие инструменты для доработки.
📁 Советы #7-9: Контроль версий
Используйте Git религиозно! Коммитьте после каждой рабочей секции. Если AI застрял в циклах, делайте git reset --hard и начинайте заново.
🧪 Советы #10-13: Тестирование
Пишите высокоуровневые интеграционные тесты, симулирующие клики пользователя. Избегайте низкоуровневых unit-тестов. Реализуйте тесты перед переходом к следующей фиче.
🐛 Советы #14-18: Отладка
Копируйте сообщения об ошибках прямо в AI. Попросите AI продумать 3-4 возможные причины перед кодированием. Сбрасывайтесь после каждой неудачной попытки.
⚙️ Советы #19-22: Продвинутая настройка
Пишите детальные файлы инструкций (100+ строк). Загружайте API документацию локально. Используйте скриншоты для показа UI багов. Попробуйте голосовой ввод для 2x быстрого промптинга.
🏗️ Советы #23-27: Архитектура
Держите файлы маленькими и модульными. Выбирайте зрелые фреймворки (Rails > Rust). Стройте сложные фичи как отдельные проекты сначала.
🚀 Советы #28-30: Не только код
Используйте AI как DevOps инженера для настройки DNS, Heroku. Генерируйте изображения, favicon'ы, графику. Используйте AI как учителя для построчного объяснения кода.
